### Conditions for Escalation:

| Scenario | Escalation Decision |
|----------|---------------------|
| User did NOT click any links | NO escalation – document and educate user |
| User clicked but firewall BLOCKED | NO escalation – threat contained |
| User clicked and firewall ALLOWED | YES escalation – potential compromise |

## Recommended Remediation Actions

### If user did NOT click:
1. Document the incident
2. Educate user about phishing awareness
3. Add malicious domain to email blacklist (if identified)

### If user clicked (check firewall logs first):
1. Force password reset for user account
2. Enable MFA immediately
3. Block malicious domain at firewall/proxy
4. Scan user workstation for malware
5. Monitor account for 30 days
